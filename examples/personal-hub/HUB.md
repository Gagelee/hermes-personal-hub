# Demo Personal Hub

Status: Public demo snapshot  
Scope: Sanitized example Hub state for demonstrating Hermes Personal Hub concepts.

## North Star

This demo shows how a private Life OS can become a public-safe snapshot without exposing personal details. It preserves the structure of active worlds, growth signals, digest surfaces, and workflow candidates while using fictional or abstract data.

## Demo Worlds

- Hermes Personal Hub: public project world for specs, visual policy, schemas, and demo renderer work.
- Fitness: synthetic training consistency surface.
- Finance: fictional review workflow candidate with protected status.
- Digest / Automations: representative recurring output surface.
- Agent Growth: public design and validator milestones.

## Safety Boundary

All data in this demo is fictional, rounded, or project-public. It does not contain real holdings, account details, raw health metrics, private paths, secrets, or local automation output.

## Product Intent

The demo should make three ideas concrete:

- Display Growth can show useful progress without custom code.
- Functional Growth can identify repeated workflows before becoming durable plugins.
- Public examples can explain the system without copying private working state.
